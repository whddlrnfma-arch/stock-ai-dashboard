# [Current Task: STEP 1] `kiwoom_logic.py` 구현
아래 요구사항을 완벽하게 반영하여 `kiwoom_logic.py`에 들어갈 클래스 및 메서드를 작성해 줘.

## 1. 진입강도 (Entry Strength) 로직
- 실시간 조건검색으로 포착된 종목의 진입강도를 계산하여 100% 기준으로 업데이트하는 함수/메서드를 작성해 줘 (구체적인 계산식은 목업(Mock-up) 처리해도 됨).
- **[필터링 규칙]:** 계산된 진입강도가 70% 미만으로 떨어지면, 해당 종목을 '조건식 이탈'로 간주하고 아래의 10분 지연 로직으로 넘길 것.

## 2. 10분 지연 (Caching) 로직
- 조건식에서 이탈하거나 진입강도가 70% 미만으로 떨어진 종목 코드를 감지했을 때, **즉시 DB에서 삭제하지 말 것.**
- `cachetools.TTLCache` (또는 Timestamp를 포함한 Dictionary)를 활용하여 이탈 시점과 함께 메모리에 캐싱(상태: `CACHED`)할 것. (TTL = 600초)
- 10분이 경과하여 캐시에서 만료되는 시점(또는 주기적으로 만료를 체크하는 루프)에만 DB에서 최종 삭제(Delete)되도록 처리 플로우를 구현할 것.

## 3. 방어적 프로그래밍 (Strict Error Handling)
- 실시간 API 특성상 들어오는 데이터가 누락되거나 타입이 맞지 않을 수 있음. 
- 데이터 파싱 및 딕셔너리 접근 시 발생할 수 있는 `KeyError`, `ValueError`, `TypeError` 등을 철저하게 `try-except` 블록으로 방어할 것.
- 예외 발생 시 시스템이 크래시되지 않고, `logging` 모듈을 사용해 에러를 기록한 뒤 스킵(`continue` 또는 `return`)하도록 짤 것.

자, 위 지침을 확실히 이해했다면 **STEP 1**에 해당하는 `kiwoom_logic.py`의 파이썬 코드만 먼저 작성해 줘.
